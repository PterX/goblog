基于飞书小龙虾“社区驱动、MCP协议、多智能体协同、记忆系统”的成功经验，AnqiCMS 完全可以站在更高的起点，直接构建一个**可对话操作的行业智能CMS**。

下面是一份具体的演进路线图，将抽象的逻辑转化为可落地的步骤。

---

### 🎯 核心策略

1.  **MCP 优先**：将所有后台功能（发文章、改模板、查数据）封装为标准的 MCP 工具，让 AI 能力可以无限扩展。
2.  **多智能体编排**：利用 Eino 的 Graph 能力，构建“计划-执行-审核”协同智能体，处理“每日生成10篇行业热点文章”这类复杂任务。
3.  **记忆系统**：深度利用 CMS 自身沉淀的行业数据（文章、标签、用户行为）作为 AI 的长时记忆，生成真正“懂行”的内容。
4.  **低门槛与社区生态**：提供可视化的 Skill 配置界面，鼓励开发者上传自定义技能，形成类似 WordPress 插件的生态系统。

---

### 🗺️ 分阶段实施路线图

#### 阶段一：MCP 化工具箱，打造“机械双钳” (4-6周)

**目标**：让 AI 能够“动手”操作 AnqiCMS，并保证安全。

*   **构建 AnqiCMS MCP Server**
    *   使用 `mcp-go` 库实现 MCP 协议，将核心 Service 层方法暴露为 MCP 工具。
    *   首批封装：文章增删改查、模板管理、栏目管理、用户管理。
    *   **示例**：`create_article`、`update_template`、`get_site_status` 等。
    *   遵循权限模型：MCP 工具操作必须校验当前用户权限，防止越权。

*   **实现 Skill 市场基础**
    *   设计 Skill 描述文件格式（如 YAML/JSON），定义 Skill 名称、描述、依赖的 MCP 工具列表和参数。
    *   允许管理员在后台通过上传文件或编写简单配置来安装新 Skill。
    *   **示例 Skill**：`热点文章生成` Skill 包含调用 `get_hot_news`、`create_article`、`validate_template_tags` 等多个工具。

*   **安全沙箱（可选但推荐）**
    *   对于模板编辑、代码生成等高风险操作，在 MCP 工具层实现语法白名单校验，拒绝可执行代码。

#### 阶段二：构建“多智能体大脑” (4-8周)

**目标**：让 AI 不仅能执行简单指令，还能自主规划并完成复杂多步任务。

*   **集成 Eino 编排引擎**
    *   在 Go 后端引入 Eino，通过 Graph 定义多智能体协同逻辑。
    *   **计划者 (Planner)**：用推理模型（如 DeepSeek-R1）分析用户复杂意图，生成任务步骤。
    *   **执行者 (Executor)**：用工具调用模型（如 GPT-4o）依次调用 MCP 工具执行步骤。
    *   **审核者 (Reviewer)**：检查执行结果，如文章质量、模板标签合规性，异常时触发重试或人工介入。

*   **实现固定工作流（Graph）**
    *   对于高频任务（如“每日10篇热点文章”），预先用 Eino Graph 编排好固定的执行流程，确保高可靠性。
    *   流程：定时触发 → 抓取行业热点 → AI选题 → 循环生成文章 → 标签/质量校验 → 批量发布。

*   **自主规划模式（可选高级功能）**
    *   针对开放性的复杂任务（如“对本栏目所有文章做一次SEO优化”），启用多智能体动态规划模式。

#### 阶段三：构建“行业记忆系统” (4-6周)

**目标**：让 AI 真正理解你的行业，生成的内容更专业、更贴合网站定位。

*   **数据知识化**
    *   将 AnqiCMS 已发布的文章、常用模板、栏目结构、标签库等数据，通过嵌入向量存入向量数据库。
    *   使用 Eino 的 `Retriever` 组件，在任务执行时检索相关历史内容、风格案例作为上下文。

*   **用户画像与偏好学习**
    *   记录用户对 AI 操作的反馈（接受/修改/拒绝），训练偏好模型，逐步让 AI 更懂你的内容风格。

*   **行业知识图谱**
    *   利用 CMS 积累的标签、关键词、栏目关系，构建简单的行业知识图谱，辅助 AI 选题和写作。

#### 阶段四：前端聊天界面与体验优化 (2-4周)

**目标**：提供飞书式对话体验，让操作透明化。

*   **开发 Chat UI 组件**
    *   在 AnqiCMS 的 React 后台中加入悬浮式或侧边栏聊天窗口。
    *   使用 SSE 流式展示 AI 的思考和执行过程（如“正在抓取热点…”、“正在生成第3篇文章…”）。

*   **任务状态可视化**
    *   对于长时间运行的任务，提供一个任务管理中心，展示进度条、日志和结果预览。
    *   支持用户中途干预（暂停、修改参数、跳过某步骤）。

*   **低代码 Skill 配置器**
    *   提供一个图形化界面，让用户可以通过拖拽 MCP 工具来创建自己的工作流（结合 Eino Dev 插件）。

### 🏁 快速启动：最小可行版本

如果你想快速验证，可以先做一个 **MVP**，包含：

1.  **后端**：实现一个 MCP Server，暴露 `create_article` 和 `list_articles` 两个工具。
2.  **AI 核心**：用 Eino 写一个最简单的 Graph，包含 LangChain 节点调用 MCP 工具。
3.  **前端**：在后台加一个聊天输入框，支持“帮我写一篇关于 XX 的文章”指令。
4.  **发布**：将此功能作为 AnqiCMS 3.x 的一个实验性插件发布，收集反馈。

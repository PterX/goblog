# 站点健康检查技能

## 概述
本技能指导如何对站点进行全面健康检查，包括系统配置、SEO 设置、安全防护、性能优化等方面。

## 使用步骤

### 第一步：检查系统配置
- 确认站点名称、关键词、描述等基本信息是否正确填写
- 检查 URL 配置（伪静态、URL 模式）
- 确认缓存设置是否合理

### 第二步：检查内容健康度
1. 使用 `archive_list` 检查：
   - 是否有大量草稿未发布（status=draft）
   - 是否有内容为空或过短的文章
   - 是否有过期内容（updated_time 很久以前）
2. 使用 `category_list` 检查是否有空分类（没有文章的分类）

### 第三步：检查 SEO 基础设置
使用相关工具检查：
- 标题/描述是否按要求填写
- 站点地图配置是否正常
- 是否有重复标题的内容

### 第四步：检查安全设置
- 确认后台登录限制是否开启
- 确认上传文件类型限制
- 确认系统已更新到最新版本

### 第五步：生成健康报告
按以下格式汇总：

```
站点健康报告 - {site_name}
生成时间：{time}

✅ 正常项：
- ...

⚠️ 警告项：
- ...

❌ 问题项：
- ...

建议优先级：
1. ...
2. ...
```

## 最佳实践
- 建议每周执行一次快速检查，每月一次全面检查
- 发现问题后立即记录，分优先级处理
- 保持系统版本更新
